# 📖 Overview

Modern AI can produce grammatically correct writing that still feels empty, repetitive, or generic. Writers often describe this phenomenon as **AI slop**.

Rather than attempting to determine whether text was generated by AI, this project focuses on identifying the **writing patterns** that make content sound artificial.

Instead of giving a vague verdict, the checker pinpoints specific problems—including filler phrases, passive voice, repetitive openings, and vague adjectives—and explains why each weakens the writing.

# 🔍 Detection Categories

| Issue | Description | Example |
|--------|-------------|----------|
| Hollow Openers | Generic introductions that delay the main point | *"In today's fast-paced world..."* |
| Filler Phrases | Empty wording that adds little meaning | *"It is important to note..."* |
| Vague Adjectives | Subjective descriptors without evidence | *"robust", "powerful"* |
| Passive Voice | Indirect sentence construction | *"The report was generated..."* |
| Throat-Clearing | Redundant transitions | *"In conclusion..."* |
| Long Sentences | 30+ word sentences that reduce readability | Automatically flagged |
| Repetitive Starters | Consecutive sentences beginning similarly | Common AI writing habit |

---

# 📊 Scoring System

Every document receives a **Slop Score (0–100).**

| Score | Rating |
|--------|---------|
| 🟢 0–15 | Clean |
| 🟡 16–35 | Minor Issues |
| 🟠 36–60 | Moderate Slop |
| 🔴 61–100 | Heavy AI Slop |

Scores are weighted by issue severity and normalized by document length to ensure fairness across both short and long documents.

# 💡 Design Decisions

<details>

<summary><strong>Why a rule-based system?</strong></summary>

This project intentionally avoids LLM APIs.

Instead of outsourcing the reasoning process, the detector explicitly defines and implements the linguistic patterns associated with AI-generated writing.

Benefits include:

- Fully transparent scoring
- Zero operating cost
- Instant execution
- Easy customization
- Works offline

</details>

<details>

<summary><strong>Why normalize by document length?</strong></summary>

Longer documents naturally contain more opportunities for issues.

Normalizing by word count ensures the score reflects issue density rather than total issue count, allowing fair comparisons across documents of different lengths.

</details>
